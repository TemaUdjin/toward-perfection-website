# plans/

Место для технических планов реализации функций. Здесь живёт информация **как** мы делаем проект.

## Правила

1. **Один план = одна функция.** Если план уже есть — работаем с ним, не плодим дубли.
2. **Имя файла:** `YYYY-MM-DD-название-функции.md` (например, `2026-04-26-paddle-integration.md`).
3. **Структура плана:** делится на фазы. У каждой фазы статус `[ ]` (не сделано) или `[x]` (сделано).
4. **Итоговый блок** в конце плана: реализован целиком или нет, что осталось, что заблокировано.
5. **Актуализация обязательна:** любой агент, работающий с планом, отмечает сделанные фазы и дописывает что именно сделал — после каждой сессии.

## Подпапки

- `sessions/` — рефлексия по сессиям (формат и правила — в `CLAUDE.md`).
